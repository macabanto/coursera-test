Section 1: Gathering some thoughts...

HTML is the structure of the website and CSS makes the website human friendly

Hyper Text Markup Language and Cascading Style Sheets: Hyper text meaning an abundance of text and Cascading meaning passed on to others.

CSS is used as a separate file as it enables the scalability of a website; instead of individually styling multiple HTML files inline/internally, doing it externally and applying it to every page means less typing/less redundant processes

Consider any website with many links; even though the content is different on each page, the style remains the same. Keeping this convention makes it easier for the user to understand what is going on without having to observe the whole page i.e: keeping navigation, main body, etc in the same area makes it so the user can expect where to look.

By doing CSS externally and linking it to multiple files, a convention is created and the website feels like a cohesive entity rather than multiple linked pages. CONSISTENCY and EFFICIENCY

In order to get CSS files linked correctly; a convention should be used within the HTML files so the CSS file can be applied to them. This means keeping id names and class names the same.
