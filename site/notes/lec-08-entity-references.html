Lecture 8 notes: HTML Character Entity References

Characters to avoid to prevent rendering issues:
  • < (less than)
  • > (greater than)
  • & (ampersand)
Basically, html interpreter recognizes special characters and must be referenced using this convention: "&X;"
I.E. for the above characters you would use:
  • &lt;
  • &gt;
  • &amp;
to display these special characters
HTML doesn't break on syntax errors: it just skips whatever it cannot comprehend

In HTML words wrap as page size is adjusted.
to keep words conjoined, use a non-breaking space: &nbsp; to provide a space between words without separating them.
For examples: Lorem ipsum dolor sit amet, consectetur adipisicing elit. Fuga officia dicta non maxime repudiandae sunt totam! Quasi praesentium nisi et!
So one key take away is the important of the space in HTML as it defines the end/start of an new string
In order to get multiple spaces between words/strings a <span> tag should be used and a margin applied
